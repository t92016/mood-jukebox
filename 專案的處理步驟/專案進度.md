## 重大技術決策
| 決策 | 理由 |
|---|---|
| 用 Netlify Functions 而非純前端 | 規劃書缺後端；Groq Key 放前端會被秒偷，必須代理 |
| 用 Netlify 而非 GitHub Pages | Pages 純靜態跑不了 Functions；Netlify 免費額度足夠且連 repo 自動部署 |
| GitHub repo 設 private | Netlify 免費版支援 private repo，兒童專案保守為上（develop skill 的 public 慣例是為了 Pages，本案不適用） |
| `Users` 集合改名 `MoodLogs` | 無帳號系統，匿名日誌用 MoodLogs 名實相符 |
| Firestore 前端直連（不經代理） | Firebase client SDK + Security Rules 本來就設計給前端用；多一層代理只增延遲 |
| Firebase 未設定時優雅降級 | `firebase-config.js` 沒填也能點歌（不寫紀錄），方便先測 UI |
| 種子歌放 `seeds/songs-seed.json` 不寫死 videoId | videoId 由 seed 腳本實際呼叫 API 取得，不假造 |
